## Installation Steps

### 1. Local Development Installation (Recommended for Testing)

In Zed:

1. Open Zed settings (`Ctrl+,`)
2. Add to `extensions.json`:

```json
{
  "extensions": {
    "msvc-cpp": {
      "version": "0.3.0",
      "git": "E:/Rust/zed-msvc-toolkit"
    }
  }
}
```

3. Restart Zed

### 2. System Requirements

- Windows 10/11
- Visual Studio 2022+ (with "Desktop development with C++" workload)
- clangd (available in PATH)
- CMake project

## Feature Verification

### V0.1-V0.2 Core Features

After opening a CMake project, the extension will:

1. **Detect MSVC environment**: Automatically find VS2022 and MSVC toolset
2. **Detect Windows SDK**: Automatically find include directories
3. **Detect compile database**: Check for `compile_commands.json`
4. **Generate `.clangd` configuration** (if it doesn't exist)

Due to current Zed API limitations, a prompt will be returned asking to manually create `.clangd`:

```
Current Zed extension API does not support writing workspace .clangd directly from the extension. Please manually create .clangd in the workspace root directory with the following content:

# Auto-generated by Zed MSVC C++ Assistant.
CompileFlags:
  Compiler: clang-cl
  Add:
    - /IC:/Program Files/Microsoft Visual Studio/2022/Community/VC/Tools/MSVC/14.40.33807/include
    - /IC:/Program Files (x86)/Windows Kits/10/Include/10.0.22621.0/ucrt
    ...
```

### Manual Configuration Steps

1. Copy the configuration content returned by Zed
2. Create `.clangd` file in the project root directory
3. Paste the configuration content
4. Reopen C/C++ files

## Known Limitations

- ✅ Cannot write directly to workspace files (Zed API limitation)
- ⚠️ CMake command execution feature requires Zed API support
- ❌ Debugging functionality (DAP) not implemented
