# Zed C++ Toolkit 安装与试用指南

本文说明如何在本地构建、安装和验证 `cpp-toolkit`。

## 1. 构建扩展

```bash
rustup target add wasm32-unknown-unknown
cargo build --target wasm32-unknown-unknown --release
```

构建产物位于：

```text
target/wasm32-unknown-unknown/release/zed_msvc_toolkit.wasm
```

> 当前 crate 名称仍是 `zed-msvc-toolkit`，因此 WASM 文件名暂时保留 `zed_msvc_toolkit.wasm`。

## 2. 本地安装

在 Zed 中使用本地扩展开发/安装方式加载仓库根目录。扩展元数据已更新为：

```toml
id = "cpp-toolkit"
name = "Zed C++ Toolkit"
```

如果手动复制扩展文件，可将以下文件放入 Zed 本地扩展目录中的同一个扩展目录：

```text
extension.toml
target/wasm32-unknown-unknown/release/zed_msvc_toolkit.wasm
languages/
```

## 3. 系统需求

基础需求：

- Zed
- `clangd`
- 与所选 preset 对应的编译器和构建工具

常见组合：

- Windows + MSVC：Visual Studio 2022+、CMake、Ninja、clangd
- GCC + CMake：gcc/g++、CMake、Ninja、clangd
- Clang + CMake：clang/clang++、CMake、Ninja、clangd
- Makefile：make、gcc/g++ 或 clang/clang++、clangd

## 4. 项目配置

在项目根目录创建：

```text
.zed/cpp-toolkit.toml
```

示例：

```toml
preset = "gcc-cmake-ninja"

[build]
build_type = "Debug"
build_dir = "build"

[run]
command = "./build/app"

[clangd]
extra_flags = ["-std=c++20"]
```

支持的 preset：

- `msvc-cmake-ninja`
- `gcc-cmake-ninja`
- `clang-cmake-ninja`
- `gcc-make`
- `clang-make`
- `custom`

未提供配置文件时，扩展会按平台选择默认 preset。

## 5. 验证功能

打开 C 或 C++ 文件后，扩展应自动：

1. 加载 `.zed/cpp-toolkit.toml`。
2. 合并 preset 与用户配置。
3. 生成或刷新自动生成的 `.clangd`。
4. 生成 `.zed/tasks.json`。
5. 启动 `cpp-toolkit-clangd`。

可检查项目根目录是否出现：

```text
.clangd
.zed/tasks.json
```

`.clangd` 顶部应包含：

```text
# Auto-generated by Zed C++ Toolkit.
```

生成的任务标签通常包括：

- `C++: Configure`
- `C++: Build`
- `C++: Clean`
- `C++: Run`

## 6. 常见问题

### 已有 `.clangd` 会被覆盖吗？

不会。用户手写的 `.clangd` 会被保留。只有旧版 `Zed MSVC C++ Assistant` 或新版 `Zed C++ Toolkit` 自动生成的 `.clangd` 会被刷新。

### 如何使用 CLion 风格构建目录？

默认构建目录是 `build`。如需 `cmake-build-debug` 这类目录，显式配置：

```toml
[build]
build_dir_style = "clion"
build_type = "Debug"
```

### 如何指定自定义 clangd？

```toml
[clangd]
command = "/opt/llvm/bin/clangd"
query_driver = ["gcc", "g++"]
```

### Windows/MSVC 还支持吗？

支持。使用：

```toml
preset = "msvc-cmake-ninja"
```

扩展会尽量发现 Visual Studio、MSVC include 和 Windows SDK include，并写入自动生成的 `.clangd`。
